## 项目定义
Trading OS = 可复制、可导航、可审计、可持续迭代的会员交易执行系统。

核心目标：
1. 会员 3 分钟进入当日执行
2. 每位学员独立副本交付，避免数据污染
3. 决策可追溯（计划/执行/复盘/风控）
4. 内容资产（大师文章/研究）持续沉淀并可检索

## 当前状态（已接管）
- 执行系统母版 v0.2 已冻结（熊市生存执行系统）
- 学员交付闭环已建立（登记表 + 复制指引）
- 大师系列已启动索引化
- 数据目录存在双 memory（根目录与仓库内）

## 本目录用途
该目录是 Trading OS 的**唯一项目管理中枢**：
- 需求、架构、流程、变更、风险、进度统一在此
- 后续推进按本目录文档为准
- 便于替补 agent 或新成员快速接管

## 目录导航
- `00-governance/`：冻结规范、变更记录、风险规则
- `01-product/`：产品形态、模块定义、用户旅程
- `02-ops/`：交付SOP、运行手册、巡检清单
- `03-data-contracts/`：数据写入规范、路径标准
- `04-execution/`：执行系统字段标准、模板规范
- `05-content-pipeline/`：大师文章索引流水线与录入标准
- `06-handover/`：交接文档、接管状态
- `07-backups/`：备份策略与恢复流程
- `08-keys-and-access/`：API/Key需求清单（不存密钥）
- `09-weekly-reports/`：周报归档
